## 핵심 개념 & 깨달음

1. **버튼별 이미지 이동 처리**

   - imgcount 값(현재 이미지 인덱스)에 따라 컨테이너의 translateX 값을 변경
   - 버튼 클릭 시 imgcount를 증/감시키며 슬라이드 위치 변환

2. **경계값(순환) 처리 실수**

   - if문 여러 분기: 코드 복잡, 확장에 약함
   - 이전/다음 "순환" 로직 헷갈림, 여러가지 케이스로 실수 발생

3. **정석 개선 패턴**

   - 증/감 후 경계값(최댓값/최솟값) 초과시 순환하도록 처리
   - 아래 코드 방식이 훨씬 효율적:

   ```js
   document.querySelector('.nextImg').addEventListener('click', function () {
   	imgcount++;
   	if (imgcount > 2) imgcount = 0;
   	imgBox.style.transform = `translateX(-${imgcount * 100}vw)`;
   });
   document.querySelector('.beforeImg').addEventListener('click', function () {
   	imgcount--;
   	if (imgcount < 0) imgcount = 2;
   	imgBox.style.transform = `translateX(-${imgcount * 100}vw)`;
   });
   ```

   - 이미지 개수 늘어날 때(3장→4~5장) 경계값만 변수로 관리하면 자동 확장 가능
