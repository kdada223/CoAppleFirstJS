# 2025-11-25 회고

## 오늘 한 강의 / 문제

- 모달창(로그인 UI)에서 검은 배경(overlay) 클릭 시 닫히게 만드는 코드, 이벤트 버블링 구조 완벽 실습
- for문 없이 탭(Products, Information, Shipping) UI를 이벤트 위임 패턴 한 방에 처리하는 법 연습

## 핵심 개념 (Pick 3)

1. **이벤트 버블링의 실제 체감**
   - 자식 요소에서 발생한 이벤트가 부모(검은배경)까지 전파
   - input, div, button 등 내부 요소를 눌러도 black-bg까지 이벤트가 이동되는 걸 실전 체감
2. **e.target과 e.currentTarget의 명확한 구별**
   - e.target: ‘유저가 실제 누른 것’, e.currentTarget: 이벤트리스너가 달린 요소
   - 모달 오버레이 클릭 시 닫기에서 가장 중요한 개념
3. **이벤트 위임과 data-속성 응용**
   - 탭이 많아도 for문 없이 상위(list)에 하나만 addEventListener!
   - data-id 속성+dataset 조합: 어떤 버튼 클릭했는지 동적 식별, 탭/버튼 갯수 확장도 자유로움

## 코드/실험 & 깨달은 점

### 1. **모달의 버블링 현상 실습**

```js
let 까만배경 = document.querySelector('.black-bg');
까만배경.addEventListener('click', function (e) {
	if (e.target == 까만배경) {
		document.querySelector('.black-bg').classList.remove('noneModal');
	}
});
```
- 검은배경만 눌렀는지 체크하는 조건이 없으면 내부 input만 클릭해도 닫히는 문제 체험
- e.target과 e.currentTarget 찍어보면, 실제 전파 과정/대상 구별이 한눈에 보임

### 2. **탭 UI를 for문 없이 이벤트 위임으로 구현**

```js
let list = document.querySelector('.list');
list.addEventListener('click', function (e) {
	if (e.target.dataset.id !== undefined) {
		tapOpen(Number(e.target.dataset.id));
	}
});
```
- 각 tab-button이 data-id로 index화, for문 없이 자유롭게 쓸 수 있음
- 탭 개수 바뀌어도 자동 반영, 코드 중복이 0됨

## 오늘의 깨달음

- DOM 구조 바꾸거나 버튼이 늘어나도, ‘이벤트 위임+data-id’ 방식이 훨씬 견고하고 확장 친화적!
- 내부/외부 요소 경계(e.target == black-bg) 체크 등 작은 차이로 실제 UX가 크게 달라진다
